# Documentation Source Files

This directory contains the **source markdown files** for the project documentation.

## Structure

```
docs_src/           # Source markdown files (edit these)
├── UDL_GUIDE.md
├── SWIFT_EXAMPLES.md
├── KOTLIN_EXAMPLES.md
├── TESTING.md
└── README.md       # This file

docs/               # Generated HTML documentation (auto-generated)
├── *.html          # HTML versions of markdown files
└── lib/            # Rust API documentation (from cargo doc)
```

## Workflow

1. **Edit markdown files** in `docs_src/`
2. **Root-level files** like `README.md`, `DEVELOPMENT.md`, and `CONTRIBUTING.md` stay in project root
3. **CI/CD automatically converts** markdown → HTML in the `docs/` folder
4. **GitHub Pages** publishes from the `docs/` folder

## Files

- **`UDL_GUIDE.md`** - UniFFI UDL interface definition guide
- **`SWIFT_EXAMPLES.md`** - Swift usage examples and integration guide
- **`KOTLIN_EXAMPLES.md`** - Kotlin/Android usage examples
- **`TESTING.md`** - Testing guide for all platforms

## Local Preview

To generate HTML locally:

```bash
# Run the documentation generation script
./scripts/doc-all.sh

# The script will:
# 1. Generate Rust API docs → docs/lib/
# 2. Convert markdown → HTML → docs/*.html

# View locally
open docs/index.html

# Or start a web server
cd docs && python3 -m http.server 8000
# Then open: http://localhost:8000/
```

## Contributing

When adding new documentation:

1. **Add markdown file** to `docs_src/`
2. **Update** [`.github/workflows/deploy-docs.yml`](../.github/workflows/deploy-docs.yml) to include the new file in the conversion list
3. **Commit both** `docs_src/*.md` and the generated `docs/*.html` files

## Auto-Generated Files

The following files in `docs/` are auto-generated and should **not** be edited directly:

- `docs/*.html` - Generated from markdown
- `docs/lib/` - Generated by `cargo doc`

Always edit the source markdown files in `docs_src/` or project root instead.
